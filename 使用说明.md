# 使用说明

## 项目包含

默认产物分为两部分：

- 主 ZIP：`BallonsOCR-windows-x64-portable-vX.X.X.zip`,包含程序本体，以及 `packs/core-models.zip`
- 可选模型包： `advanced-ocr.zip`

在大多数情况下，主ZIP里面的内容已经可以满足使用，如果有其他需要，请下载`advanced-ocr.zip`，在文件夹内进行解压后重启项目

无论如何，请点击`BallonsOCR 启动.bat`，等待后即可使用

## 关于模型

1. 软件启动时，会扫描一次本地已经有的模型。
2. 设置页和底部下拉框，显示这次扫描到的可用模型。
3. 你后来新加了模型，必须重启软件，新的模型才会显示出来。

## 默认能直接用什么

正常解压主包后，不额外下载任何东西，默认直接使：

| 类型 | 默认可用项 | 说明 |
| --- | --- | --- |
| 文本检测 | `ctd` | 默认文本检测模型 |
| OCR | `smart_ocr` | 默认 OCR 分发器 |
| OCR 实际主力 | `mit48px` | `smart_ocr` 默认主要调用它 |

### `smart_ocr` 到底是什么

`smart_ocr` 不是单独的模型，它更像“自动分流器”：

- 遇到竖排文本，优先交给 `vertical_ocr`
- 遇到横排文本，优先交给 `horizontal_ocr`
- 如果第一次识别失败，或者结果是空的，再交给 `fallback_ocr`

当前默认打包配置里，`smart_ocr` 的默认路线是：

| 项目 | 默认值 |
| --- | --- |
| `vertical_ocr` | `mit48px` |
| `horizontal_ocr` | `mit48px` |
| `fallback_ocr` | `mit48px` |

## 怎样添加更多模型

两种方式：

1. 添加模型压缩包
2. 直接添加模型文件

下面按最简单的方法说。

## 方法一：添加 `advanced-ocr.zip`

这是最适合普通用户的方法。

### 这一步会增加什么

加入 `advanced-ocr.zip` 之后，通常可以增加这些模型：

| 模型 | 说明 |
| --- | --- |
| `mit32px` | 可选 OCR |
| `mit48px_ctc` | 可选 OCR |
| `manga_ocr` | 可选 OCR |
| `PaddleOCRVLManga` | 可选 OCR |

### 操作步骤

1. 先关闭软件。
2. 找到你的 BallonsOCR 主文件夹。
3. 把 `advanced-ocr.zip` 放到下面任意一个位置：

```text
BallonsOCR/packs/
```

或者：

```text
BallonsOCR/optional-packs/
```

4. 重新启动软件。
5. 打开设置页，或者看窗口下方的 OCR 下拉框。
6. 如果新模型已经出现，说明识别到了。
7. 第一次真正使用这些模型时，软件可能会提示你解压模型包，按提示确认即可。或者你也可以手动解压

### 如果重启后还是没显示

按这个顺序检查：

1. 文件名是不是就叫 `advanced-ocr.zip`
2. 压缩包是不是放在 `packs` 或 `optional-packs` 里面
3. 你是不是加完压缩包后没有重启软件

## 方法二：直接放模型文件

这种方式适合你已经有单独模型文件，不想用压缩包。

### 1. 添加 `ysgyolo`

把你的 YOLO 检测模型放到：

```text
BallonsOCR/data/models/
```

文件名建议类似下面这样：

```text
ysgyolo_xxx.pt
ultralyticsyolo_xxx.pt
```

支持的常见后缀：

- `.pt`
- `.ckpt`
- `.pth`
- `.safetensors`

放好后：

1. 关闭软件
2. 放入文件
3. 重新打开软件
4. 看“文本检测”下拉框里是否出现 `ysgyolo`

### 2. 添加 `one_ocr`

把下面两个文件放到：

```text
BallonsOCR/data/models/one-ocr/
```

必须有这两个文件：

- `oneocr.dll`
- `oneocr.onemodel`

放好后重启软件，再看 OCR 下拉框里是否出现 `one_ocr`。

### 3. 添加 `mit48px_ctc`

如果你不是用 `advanced-ocr.zip`，而是手动放文件，那么至少要有：

```text
BallonsOCR/data/models/mit48pxctc_ocr.ckpt
BallonsOCR/data/alphabet-all-v5.txt
```

放好后重启软件，再看 OCR 下拉框里是否出现 `mit48px_ctc`。

### 4. 添加 `mit32px`

需要这个文件：

```text
BallonsOCR/data/models/mit32px_ocr.ckpt
```

放好后重启软件，再看 OCR 下拉框里是否出现 `mit32px`。

### 5. 添加 `manga_ocr`

需要这个目录：

```text
BallonsOCR/data/models/manga-ocr-base/
```

目录里至少要有模型和配置文件，不能只有一个空文件夹。

### 6. 添加 `PaddleOCRVLManga`

需要这个目录：

```text
BallonsOCR/data/models/PaddleOCR-VL-For-Manga/
```

同样，目录里要有完整模型文件，不能只有一个空目录。

## 添加后怎么看有没有成功

最简单的判断方法：

1. 重启软件
2. 打开设置页
3. 查看“文本检测”和“文字识别”的下拉框

如果能看到新模型名字，通常就说明扫描成功了。

你也可以看主界面底部：

- `文本检测`
- `文字识别`

这两个下拉框也会同步显示已扫描到的可用项。

## 推荐用法

如果你不想折腾，直接这样用：

1. 保持默认 `ctd`
2. 保持默认 `smart_ocr`
3. 先实际跑一遍看看效果
4. 只有在默认效果不够时，再加 `advanced-ocr.zip`
