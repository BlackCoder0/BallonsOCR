# 使用说明

## 项目包含

默认产物分为两部分：

- 主 ZIP：`BallonsOCR-windows-x64-portable-vX.X.X.7z`,包含程序本体，以及 `packs/core-models.zip`
- 可选模型包： `advanced-ocr.zip`

在大多数情况下，主ZIP里面的内容已经可以满足使用，如果有其他需要，请下载`advanced-ocr.zip`，把它放到程序目录下的 `packs/` 或 `optional-packs/` 后重启项目。通常不要在程序根目录手动覆盖解压这个压缩包。

无论如何，请点击`BallonsOCR 启动.bat`，等待后即可使用

## 关于模型

1. 软件启动时，会扫描一次本地已经有的模型。
2. 设置页和底部下拉框，显示这次扫描到的可用模型。
3. 你后来新加了模型，必须重启软件，新的模型才会显示出来。

## 默认能直接用什么

正常解压主包后，不额外下载任何东西，默认直接使：

| 类型 | 默认可用项 | 说明 |
| --- | --- | --- |
| 文本检测 | `ctd` | 默认文本检测模型 |
| OCR | `smart_ocr` | 默认 OCR 分发器 |
| OCR 实际主力 | `mit48px` | `smart_ocr` 默认主要调用它 |

### `smart_ocr` 到底是什么

`smart_ocr` 不是单独的模型，它更像“自动分流器”：

- 遇到竖排文本，优先交给 `vertical_ocr`
- 遇到横排文本，优先交给 `horizontal_ocr`
- 如果第一次识别失败，或者结果是空的，再交给 `fallback_ocr`

当前默认打包配置里，`smart_ocr` 的默认路线是：

| 项目 | 默认值 |
| --- | --- |
| `vertical_ocr` | `mit48px` |
| `horizontal_ocr` | `mit48px` |
| `fallback_ocr` | `mit48px` |

## 怎样添加更多模型

两种方式：

1. 添加模型压缩包
2. 直接添加模型文件

### 方法一：添加 `advanced-ocr.zip`

这是最适合普通用户的方法。

#### 这一步会增加什么

加入 `advanced-ocr.zip` 之后，通常可以增加这些模型：

| 模型 | 说明 |
| --- | --- |
| `mit32px` | 可选 OCR |
| `manga_ocr` | 可选 OCR |
| `PaddleOCRVLManga` | 可选 OCR |

实际显示哪些模型，以压缩包里真实包含的文件为准。

#### 操作步骤

1. 先关闭软件。
2. 找到你的 BallonsOCR 主文件夹。
3. 把 `advanced-ocr.zip` 放到下面任意一个位置：

```text
BallonsOCR/packs/
```

或者：

```text
BallonsOCR/optional-packs/
```
个人推荐放到`BallonsOCR/optional-packs/`。这样主包和可选包更容易区分，也方便后续替换。

4. 重新启动软件。
5. 打开设置页，或者看窗口下方的 OCR 下拉框。
6. 如果新模型已经出现，说明识别到了。
7. 第一次真正使用这些模型时，软件可能会提示你解压模型包，按提示确认即可。
8. 如果你已经把压缩包放进去了，就不要再把它手动覆盖解压到程序根目录，否则容易把文件放乱。

#### 如果重启后还是没显示

按这个顺序检查：

1. 文件名是不是就叫 `advanced-ocr.zip`
2. 压缩包是不是放在 `packs` 或 `optional-packs` 里面
3. 你是不是加完压缩包后没有重启软件
4. 压缩包本身是不是完整；如果缺少模型文件，那么对应模型不会显示



### 方法二：手动增加第三方模型

这里说的“第三方模型”，更准确地说，是把外部拿到的模型文件接入到 BallonsOCR 已经支持的后端里。不是任意新模型结构丢进来都能直接识别。

#### 先判断属于哪一类

常见有两种：

1. 第三方检测模型
2. 第三方 OCR 模型

#### 第三方检测模型怎么加

目前最容易接入的是 `ysgyolo` 这一类 YOLO 检测模型。

你可以把文件放到：

```text
BallonsOCR/data/models/
```

文件名建议以这些前缀开头：

```text
ysgyolo_xxx.pt
ultralyticsyolo_xxx.pt
```

支持的常见后缀：

- `.pt`
- `.ckpt`
- `.pth`
- `.safetensors`

放好后重启程序，再去“文本检测”下拉框里看是否出现 `ysgyolo`。

如果出现了，但默认没选中，可以在设置页手动切换到它。

#### 第三方 OCR 模型怎么加

目前程序支持的方式，主要是把第三方模型整理成某个已支持 OCR 后端要求的目录结构。

例如：

- `one_ocr` 需要 `oneocr.dll` 和 `oneocr.onemodel`
- `mit32px` 需要 `mit32px_ocr.ckpt`
- `manga_ocr` 需要完整的 `manga-ocr-base/` 目录
- `PaddleOCRVLManga` 需要完整的 `PaddleOCR-VL-For-Manga/` 目录

也就是说，如果你拿到的是第三方来源的这些模型文件，只要文件名、目录名、放置路径都符合要求，程序就能把它们识别成对应后端。

#### 不支持“任意未知模型直接丢进去”

如果一个第三方模型：

- 不是上述这些已支持后端之一
- 文件名和目录结构完全不同
- 还需要额外的推理代码、预处理、后处理

那么它不会因为“放进 `data/models/`”就自动生效。这种情况需要额外开发适配代码，不属于单纯“拷贝模型文件”能解决的范围。

#### 第三方模型接入建议

为了尽量少踩坑，建议按这个顺序做：

1. 先确认 BallonsOCR 里有没有对应的后端名字。
2. 再确认这个后端要求的文件名和目录结构。
3. 把模型放到正确位置。
4. 重启程序。
5. 到设置页查看下拉框里有没有出现对应模型。
6. 先用一张样例图测试，再决定是否长期使用。

如果只是想增加更多 OCR，一般优先推荐：

1. 先用 `advanced-ocr.zip`
2. 如果里面没有你要的模型，再手动按对应结构放文件
3. 只有在程序本身不支持该模型类型时，才考虑后续开发适配

#### 添加后怎么看有没有成功

最简单的判断方法：

1. 重启软件
2. 打开设置页
3. 查看“文本检测”和“文字识别”的下拉框

如果能看到新模型名字，通常就说明扫描成功了。

你也可以看主界面底部：

- `文本检测`
- `文字识别`

这两个下拉框也会同步显示已扫描到的可用项。

## 推荐用法

如果你不想折腾，直接这样用：

1. 保持默认 `ctd`
2. 保持默认 `smart_ocr`
3. 先实际跑一遍看看效果
4. 只有在默认效果不够时，再加 `advanced-ocr.zip`
